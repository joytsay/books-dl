# 博客來電子書下載解密指南

## 概述

本文件記錄從博客來（books.com.tw）下載已購買 EPUB 電子書，並解除其 `enc01` DRM 加密的完整流程。

核心原理：讀取器頁面（viewer-ebook.books.com.tw）採用 **XOR + SHA256 金鑰推導** 的方式進行串流加密，每個檔案的金鑰由 **download_token** 與 **檔案路徑** 共同決定。

## 快速開始（最簡路徑）

### 前置條件

- 已從博客來購買該電子書
- Edge 瀏覽器已登入博客來並開啟書櫃頁面
- Python 3.8+（需安裝 `requests`、`openpyxl`）

### 步驟 A：取得 Cookie

1. 用 Edge 開啟 `https://viewer-ebook.books.com.tw/viewer/index.html?readlist=all`
2. 按 F12 → **Application** → Cookies → `viewer-ebook.books.com.tw`
3. 複製以下 4 個 Cookie 值：
   - `CmsToken`
   - `DownloadToken`
   - `bid`
   - `ssid`

### 步驟 B：填入 Cookie

編輯 `books_config.json`（僅需 cookies，books 陣列可空，output 區段可自訂目錄）：

```json
{
  "cookies": {
    "CmsToken": "步驟 A 複製的值",
    "DownloadToken": "步驟 A 複製的值",
    "bid": "步驟 A 複製的值",
    "ssid": "步驟 A 複製的值"
  },
  "output": {
    "download_dir": "downloads",   /* EPUB 輸出目錄（相對或絕對路徑） */
    "export_dir": "."               /* Excel 匯出目錄（相對或絕對路徑） */
  },
  "books": []
}
```

> `output` 區段可省略，預設值即為 `downloads` 和 `.`。

### 步驟 C：執行全自動批次下載

```bash
cd C:\projects\books_dl
python books_dl_v2.py
```

腳本會自動：
1. 匯出書架 Excel（207 本）
2. 過濾出「❌ 未下載」**和「🆕 新購買」** 的 reflowable 書
3. 批次下載（自動追蹤進度）
4. 更新 Excel 標註
5. 再下載「未下載 + 新購買」的 fixedlayout 書
6. 最終更新 Excel

輸出在 `downloads/` 目錄，檔名自動為 `書名 - 作者.epub`。

## 詳細 API 說明

### 下載 API: BookDownLoadURLII

**端點：** `GET https://appapi-ebook.books.com.tw/V1.7/CMSAPIApp/BookDownLoadURLII`

| 參數 | 範例值 | 說明 |
|------|--------|------|
| `book_uni_id` | `E050134228_reflowable_normal` | 書籍統一 ID |
| `t` | `1781577276168` | 時間戳（毫秒） |

**注意：** 原始的 `BookDownLoadURL` 已失效（回傳 `id_err_303`），請改用 `BookDownLoadURLII`。

**回應範例：**

```json
{
  "book_uni_id": "E050134228_reflowable_normal",
  "download_link": "https://streaming-ebook.books.com.tw/V1.0/Streaming/bookII/E55837/9897747/",
  "download_token": "77b642+ali3/...（約 960 字元）",
  "size": 1638349,
  "encrypt_type": "enc01",
  "drm_type": "1"
}
```

### 書架 API: ReadList

**端點：** `POST https://appapi-ebook.books.com.tw/V1.7/CMSAPIApp/ReadList`

| 參數 | 說明 |
|------|------|
| `offset` | 分頁偏移（0） |
| `page_size` | 每頁數量（40） |
| `sort_order` | `TimeDesc`（依購買日期排序） |
| `listname` | `["all","trial"]` |
| `cat` | `E05` |

支援 207 本書分頁讀取，回傳 `total_records`、`records`（每筆含 `book_uni_id`、`c_title`、`author`、`auth_time`、`isbn` 等）。

## 解密核心演算法

### enc01 是 XOR + SHA256，不是 AES

#### 1. 提取檔案路徑

從完整 streaming URL 中提取（**必須先剝除 query string**）：

```python
url_clean = url.split("?")[0]
re.search(r"https?://(?:.*?/){3}.*?(/.+)", url_clean)
# 範例輸入:
#   https://streaming-ebook.books.com.tw/V1.0/Streaming/bookII/E55837/9897747/META-INF/container.xml?DownloadToken=...
# 範例輸出:
#   /E55837/9897747/META-INF/container.xml
```

#### 2. 計算 partition 值（MD5 分組求和 mod 64）

```python
md5_hex = MD5(file_path)                          # 32 字元 hex
partition = sum(int(md5_hex[i:i+4], 16) for i in range(0, 32, 4)) % 64
```

#### 3. 產生 32-byte 金鑰

```python
key_string = download_token[:partition] + file_path + download_token[partition:]
key = SHA256(key_string)      # 32 bytes
```

#### 4. XOR 解密

```python
decrypted[i] = encrypted[i] XOR key[i % 32]
```

**BOM 處理**：若解密後前 3 bytes 為 `0xEF 0xBB 0xBF`（UTF-8 BOM），將其移除。

### 檔名清理（sanitize）

書名/作者可能包含 Windows 無法作為檔案名稱的字元（如 `\r\n`、`\t`、`<>:"/\|?*` 等），需先清理：

```python
def sanitize(s):
    s = s.strip()
    s = s.replace('\r', '').replace('\n', '').replace('\t', '')
    s = re.sub(r'[<>:"/\\|?*]', '_', s)
    return s.strip('_. ')
```

### YA() Token：資源檔專用

container.xml 和 content.opf 使用 raw `download_token` 下載；其他資源必須使用 YA() 產生的 HMAC token：

```python
def ya_token(token, path):
    k = str(round(67 * sqrt(len(path)))) + path
    msg = f"{token}|{path}|web"
    return HMAC_SHA256(msg, k).hexdigest() + token
```

`path` 為資源相對路徑（如 `OEBPS/story-1.xhtml`）。

| 檔案類型 | URL token | 需解密？ |
|---------|-----------|---------|
| `container.xml`, `content.opf` | raw `download_token` | 是 |
| `.xhtml`, `.css`, `.jpg`, `.ncx`, `.mp3` | YA(token, path) | 是（音檔不解密） |
| `.ttf`, `.woff` 等字型 | 無需 token | 否 |

音檔（`.mp3`, `.m4a`, `.wav`）使用 YA token 下載但**不作 XOR 解密**，直接存為 binary。

## 批次下載說明

### 輸出目錄設定

可透過 `books_config.json` 的 `output` 區段自訂目錄：

| 欄位 | 預設值 | 說明 |
|------|--------|------|
| `download_dir` | `downloads` | EPUB 下載目錄（可設相對或絕對路徑，如 `D:/我的書櫃`） |
| `export_dir` | `.`（專案根目錄） | Excel 書架匯出目錄 |

相對路徑以專案根目錄或 exe 所在目錄為基準。若該區段不存在，自動使用預設值。

### 編譯版執行檔

提供 `.exe` 編譯版本，無需安裝 Python 即可執行：

```bash
# 確保 books_config.json 在同目錄
雙擊 books_dl.exe
```

exe 模式使用 `get_base_dir()` 函式自動偵測執行檔位置（而非 `__file__` 暫存路徑），確保所有相對路徑解析正確。

### 自動書單模式（預設）

`books_dl_v2.py` 現已全自動化：

1. 呼叫 `export_shelf.py` → 產出最新書架 Excel
2. 自動讀取 Excel，用 `extract_pending_ids()` 同時過濾「❌ 未下載」**和「🆕 新購買」** 的書
3. **先下載 reflowable**（純文字，通常較小，速度快）
4. 更新 Excel 標註已下載
5. **再下載 fixedlayout**（圖文書，較大）——同樣過濾「未下載 + 新購買」
6. 最終更新 Excel

> **「新購買」也會自動下載**：剛買的書標記為 🆕 新購買，腳本會自動帶入 `book_uni_id` 並下載，不需手動操作。

不需手動編輯 `books_config.json` 中的 `books` 陣列，腳本會自動填寫。

### 手動書單模式

若只想下載特定幾本書，可在 `books_config.json` 中填入 `books` 陣列，腳本會略過自動書單提取，直接使用手動清單。

### Cookie 時效

- Cookie 有效期約 **30 分鐘**
- 期間可下載約 40-80 本小型 reflowable 書
- 若中途過期：重新從瀏覽器取得 Cookie → 更新 `books_config.json` → 再執行
- 已下載的書透過 `downloads/.downloaded.json` 追蹤，**不會重複下載**

### 下載追蹤機制

每成功下載一本書，自動寫入 `downloads/.downloaded.json`：

```json
{
  "E050134228_reflowable_normal": {
    "file": "比特幣大騙局 - 作者.epub",
    "title": "比特幣大騙局"
  }
}
```

`export_shelf.py` 讀取此追蹤檔優先比對 `book_uni_id`，零誤判。

## 書架匯出工具（export_shelf.py）

### 功能

- 從 `ReadList` API 取得所有書
- 依購買日期（舊→新）排序
- 比對下載狀態（追蹤檔優先，檔名備援）
- **自動 NotebookLM 分類**（第 10 欄 + 第二工作表「分類統計」）
- 輸出 Excel 含時間戳（每次執行產生新備份）

### 狀態標註

| 狀態 | 條件 |
|------|------|
| ✅ 已下載 | `book_uni_id` 在追蹤檔中，或檔名比對相符 |
| ❌ 未下載 | 購買日期早於今天，未在 `downloads/` |
| 🆕 新購買 | 今天購買，尚未下載（分類欄強制顯示「未分類」） |

### Excel 欄位

| 欄位 | 說明 |
|------|------|
| 序號 | 排序編號 |
| 狀態 | ✅ 已下載 / ❌ 未下載 / 🆕 新購買 |
| book_uni_id | 用於下載的 ID |
| 書名 / 作者 / 出版社 | 書籍資訊 |
| ISBN | 國際標準書號 |
| 購買日期 / 出版日期 | 日期資訊 |
| **NotebookLM 分類** | 自動分類（含第二工作表「分類統計」） |

### 執行

```bash
python export_shelf.py                          # 使用預設帳號規則
python export_shelf.py --account work           # 使用指定帳號的分類規則
```

### 分類機制

- **優先**：讀取 `category_rules.json`（外部 JSON，10 大分類 × 數百關鍵字）
- **備援**：內建 `_DEFAULT_RULES` + `_DEFAULT_FALLBACKS`
- **短關鍵字安全機制**：≤3 字母 ASCII 關鍵字使用 word-boundary regex 比對，避免 `ETF` 誤匹配 `Netflix`
- **新購買**：狀態為 🆕 新購買的書，分類欄強制「未分類」
- 書名先去空白轉小寫後做子字串比對

## Google Drive 上傳（upload_to_drive.py）

### 功能

- 掃描 `downloads/` 目錄的 EPUB → 比對 Drive 目標資料夾 → 只上傳新檔案
- 支援多帳號，各自獨立資料夾、獨立 OAuth 憑證
- 自動 backup/swap/restore auth token 與 client_secret

### 執行

```bash
python upload_to_drive.py --account BBB        # 上傳到 BBB 的資料夾
python upload_to_drive.py --account AAA       # 上傳到 AAA 的資料夾
python upload_to_drive.py                           # 使用預設帳號
```

### 首次登入

```bash
python upload_to_drive.py --login BBB
python upload_to_drive.py --login AAA
```

> **注意**：瀏覽器跳出 OAuth 畫面時，務必確認選取的帳號與 `--login` 指定的名稱一致。瀏覽器會自動選取當前已登入的帳號，若需切換請點「使用其他帳戶」。

### cp950 日文字元陷阱

含日文字元的 EPUB 檔名（如 `即効!耳ほぐし` 中的 `効`）在 Windows cp950 編碼下無法列印，會導致 `subprocess` 的 `text=True` 爆 `UnicodeEncodeError`。

**解法**：
1. `subprocess.run()` 使用 `capture_output=True` 不加 `text=True`，改為 `stdout.decode("utf-8", errors="replace")`
2. 設環境變數 `env={"PYTHONIOENCODING": "utf-8"}`，避免 `google_api.py` 內部 print 時也炸

```python
env = os.environ.copy()
env["PYTHONIOENCODING"] = "utf-8"
result = subprocess.run(cmd, capture_output=True, timeout=300, env=env)
stdout = result.stdout.decode("utf-8", errors="replace")
```

## 常見問題

### Q1: 為什麼 download_token 每次不同？

每次呼叫 `BookDownLoadURLII` 都會產生新的 session token，這是正常的 session 機制。

### Q2: 解密後的檔案無法開啟？

- 確認 `mimetype` 內容為 `application/epub+zip`（無換行）
- 確認所有檔案都成功下載（沒有遺漏）
- 確認 content.opf 中的路徑與實際檔案路徑一致
- 確認 `extract_file_path()` 先剝除了 query string

### Q3: Cookie 在哪裡取得？

1. 用 Edge 開啟 `https://viewer-ebook.books.com.tw/viewer/index.html?readlist=all`
2. 按 F12 → **Application** → **Cookies** → `viewer-ebook.books.com.tw`
3. 複製 `CmsToken`、`DownloadToken`、`bid`、`ssid`

### Q4: 如何找到 book_uni_id？

自動模式不需要手動找。若需手動查詢：
- 從 viewer URL：`https://viewer-ebook.../?book_uni_id=E050129267_reflowable_normal&ran=...`
- 或使用 `export_shelf.py` 匯出整份書架 Excel

### Q5: 音檔（MP3）下載失敗？

已在 `YA_EXTS` 和 `BINARY_EXTS` 中加入 `.mp3`、`.m4a`、`.wav`、`.ogg`，會自動使用 YA token 下載並以二進位格式儲存。

### Q6: Cookie 過期了怎麼辦？

重新從瀏覽器取得 Cookie（步驟 A），更新 `books_config.json` 中的對應值，再執行即可。已下載的書不會重複下載。
